ISMS Security Hardening & Vulnerability Remediation

This page documents the work I led while preparing for and responding to the KISA(Korea Internet & Security Agency) ISMS security audit. I proactively reviewed existing authentication, access control, and personal data handling mechanisms, identified security gaps raised during the audit, and implemented improvements to meet ISMS requirements without compromising operational usability.

Key Improvements

1. Authentication & Authorization

  • Enforced role-based access control for sensitive operations (e.g., settlement file downloads)
  • Introduced token expiration policies and tightened session handling
  • Modified authentication flow to issue access tokens only after successful OTP verification

2. Personal Data Protection

  • Encrypted or masked sensitive administrator information (name, phone number, OTP codes)
  • Separated personal data into dedicated columns for controlled access
  • Improved data retention and deletion policies for terminated accounts

3. Auditability & Traceability

  • Enhanced admin action logs for password changes and sensitive operations
  • Improved history tracking for admin creation, update, and deletion events
  • Ensured security-relevant actions are traceable for audit purposes

Outcome

  • Successfully passed ISMS security audit
  • Reduced exposure of sensitive data
  • Improved security posture and operational confidence
