ISMS Security Hardening & Vulnerability Remediation

This page describes a major framework upgrade where I migrated a production system from Spring Boot 2.x to 3.x to address security risks and improve maintainability.

Context

  • Legacy system running on Spring Boot 2.x and Java 8
  • Required to address security vulnerabilities and long-term maintainability
  • Major breaking changes introduced in Spring Boot 3.x

Key Migration Challenges & Decisions

1. Jakarta EE Migration

  • Updated all `javax.*` references to `jakarta.*`
  • Verified framework and library compatibility across the codebase

Java 17 Upgrade

  • Migrated runtime from Java 8 to 17
  • Adopted modern language features (e.g., text blocks) where readability improved

ORM & Persistence Changes

  • Addressed Hibernate 6 behavior changes and removed deprecated configuration
  • Ensured ID generation and persistence behavior remained consistent

Infrastructure & Runtime Behavior

  • Explicitly configured shutdown behavior due to changes in Spring Boot 3 defaults
  • Reviewed HTTP client usage and clarified HTTP/2 configuration assumptions

Test & Tooling Adjustments

  • Adapted test configuration due to deprecation of @MockBean and @SpyBean
